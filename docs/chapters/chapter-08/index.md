# 第8章：ADR（意思決定記録：背景→選択肢→決定→影響）

## この章で学ぶこと

- ADR は「意思決定の再現性」のために残す
- 背景→選択肢→決定→影響→移行で整理する
- 決定理由と未決事項を分ける

## この章の成果物（または判断基準）

- ADR（付録テンプレ: [ADRテンプレ](../../appendices/templates/adr/)）
- 判断の根拠と影響範囲

## 本文

本章は、テンプレ→記入例→落とし穴→チェックリストの順で整理する。

### テンプレ

- 付録: [ADRテンプレ](../../appendices/templates/adr/)

### 記入例（要点）

- 背景: 何が課題か
- 選択肢: A/B/C
- 決定: 採用案と理由
- 影響: 運用/互換性/セキュリティ
- 移行: いつ/どう/ロールバック

### よくある落とし穴

- 選択肢が書かれていない
- 影響（運用負荷）が抜ける
- 移行/ロールバックが無い

## 具体例（悪い例→良い例）

### 悪い例

```md
決定: A にする
理由: 記載なし
影響: 記載なし
```

### 良い例

```md
背景: 誤検知が多く一次対応が逼迫
選択肢: A=閾値調整 / B=SLI再設計
決定: B（説明責任と誤検知削減）
影響: ダッシュボード/アラート定義変更
移行: 段階導入、失敗時は旧定義へ戻す
```

## チェックリスト

- [ ] 背景と課題が書かれている
- [ ] 選択肢が列挙されている
- [ ] 決定理由が書かれている
- [ ] 影響（運用/互換性）が書かれている
- [ ] 移行/ロールバックがある

## まとめ

- ADR は背景・選択肢・決定理由・影響・移行/ロールバックを一続きで記録する
- 決定理由と未決事項を分離し、将来の再検討コストを下げる
