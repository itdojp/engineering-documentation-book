# 第2章：読者と目的から逆算する（Who/Why/What）

## この章で学ぶこと

- 読者（運用者/開発者/監査/顧客）を特定する
- 目的（理解/実行/判断）を固定する
- Who/Why/What を短文で定義する

## この章の成果物（または判断基準）

- 読者×目的の整理（Who/Why/What）
- ドキュメントの成功条件（読者が何を出来れば良いか）

## 本文

同じ内容でも、読者が違えば必要な情報は違う。読者を決めない文書は“誰にも刺さらない”。

### Who/Why/What

- Who: 誰が使うか
- Why: 何のために使うか
- What: 何が分かる/できるようになるか

この3点を冒頭に置くと、スコープが暴れにくい。

## 具体例（悪い例→良い例）

### 悪い例

```md
対象: みんな
目的: 理解してもらう
（何をどう理解するかが曖昧）
```

### 良い例

```md
Who: 運用当番（オンコール）
Why: アラート発生時に初動を再現する
What: 確認順・切り分け・エスカレーション基準が分かる
```

## チェックリスト

- [ ] Who/Why/What が明記されている
- [ ] 成功条件（読者が出来ること）がある
- [ ] 非スコープが暗黙になっていない

## まとめ

- ドキュメントは成果物であり、判断基準と運用で品質が決まる
- テンプレとチェックリストを“運用に組み込む”ことで事故を減らす
