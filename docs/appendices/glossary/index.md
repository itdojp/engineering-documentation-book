# 用語集

- ADR: Architecture Decision Record（意思決定記録）
- Docs-as-Code: ドキュメントをコードと同様に版管理・レビュー・CI で運用する考え方
- IC: Incident Commander（インシデントコマンダー）
- Owner: 更新責任者（ドキュメントの棚卸し・更新を担う担当）
- Runbook: 異常時を含む運用手順書
- SLI: Service Level Indicator（サービスレベル指標）
- SLO: Service Level Objective（サービスレベル目標）
- SLA: Service Level Agreement（サービスレベル合意）
- Mermaid: テキストで図（構成/フロー/シーケンス等）を表現する記法
- SOP: Standard Operating Procedure（標準手順書）
- RACI: 役割分担（責任者/実行者/相談先/報告先）を明確化する枠組み
- オンコール: 障害/アラート対応の当番（on-call）
- ロールバック: 変更を元に戻して影響を抑える復旧方針
- ロールフォワード: 追加の修正で前に進めて復旧する方針
- 責任分界: 誰がどこまで責任を負うか（境界/前提/例外）を明確化すること
- インシデント: サービスに影響する事象（障害/セキュリティ事故等）の総称
- ポストモーテム: 障害/事故の振り返り（責任追及しない）
