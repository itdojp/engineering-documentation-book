# チェックリスト集

## ドキュメントレビュー観点

- [ ] 読者（誰が使うか）が明記されている
- [ ] 目的（何のために書くか）が明記されている
- [ ] 前提（環境/権限/依存）が明記されている
- [ ] 手順の再現性（成功条件/検証）がある
- [ ] 例外系・失敗時の分岐がある
- [ ] ロールバック判断がある
- [ ] 責任分界（誰が何をするか）がある

## Runbook 完備性チェック

- [ ] 平常時の作業が定義されている
- [ ] アラートの初動（確認順）が定義されている
- [ ] 切り分けの観点（比較/変更点）がある
- [ ] エスカレーション基準がある
- [ ] 復旧/ロールバック判断がある

## ポストモーテム品質チェック

- [ ] 個人攻撃になっていない（責任追及しない）
- [ ] 真因が“仕組み”として説明されている
- [ ] 再発防止が行動（誰が/いつまでに/何を）になっている
- [ ] 検知・手順の改善に反映されている
