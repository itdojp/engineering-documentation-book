# 手順書テンプレ

実行前提→実行→検証→失敗時→ロールバックの順で、再現性を担保します。

## コピペ用テンプレ

## 目的

- 何を、どの状態にする手順か

## 入力（前提）

- 対象環境（例: production / staging）
- 必要権限
- 依存（停止可否、メンテ枠）

## 手順

1. 準備
2. 実行
3. 検証（成功条件）

## 成功条件（確認観点）の例

- 外形監視: `GET /healthz` が `200` を返す、主要画面が表示される
- メトリクス: エラー率が基準値未満、レイテンシ p95 が閾値未満
- ログ: 重大エラーが増加していない（特定のエラーコード/例外が出ていない）

## 例外/失敗時

- 失敗パターンと判断基準
- 連絡先/エスカレーション

## ロールバック

- どこまで戻すか
- 影響と注意点

## 責任分界

- 実行者
- 承認者
- 影響を受ける利用者
