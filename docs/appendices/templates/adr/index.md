# ADR テンプレ

意思決定の背景と選択肢を残し、将来の再検討コストを下げます。

## コピペ用テンプレ

```md
## 背景

- なぜ今この判断が必要か

## 課題

- 現状の問題
- 制約（コスト/期限/運用）

## 選択肢

- 選択肢A
- 選択肢B
- 選択肢C

## 決定

- 採用する案
- 採用理由

## 影響

- 影響範囲
- 運用負荷
- セキュリティ/プライバシー

## 移行

- いつ/どう移行するか
- ロールバック

## 関連リンク

- Issue/PR
- 参考資料
```

## 記入例（最小）

### 背景

- 監視アラートが多く、一次対応の負荷が高い

### 課題

- 既存の閾値は根拠が薄く、誤検知が多い

### 選択肢

- A: 既存の閾値を調整する
- B: サービスレベル指標（SLI）で再設計する

### 決定

- B を採用する（誤検知削減と説明責任を両立）

### 影響

- ダッシュボード/アラート定義の変更が必要

### 移行

- 段階導入（既存アラートと並行稼働し、誤検知/検知漏れを比較する）
- ロールバック: 旧アラート定義に戻せるよう、設定と手順を残す

### 関連リンク

- Issue: `#123`（例）
- PR: `#456`（例）
- 参考: SLI/SLO の設計メモ（社内Wiki等）

## 具体例（悪い例→良い例）

### 悪い例

```md
決定: B にする
理由: なんとなく
影響: なし
```

### 良い例

```md
背景: 誤検知が多く一次対応が逼迫
選択肢: A=閾値調整 / B=SLI再設計
決定: B（説明責任と誤検知削減を両立）
影響: ダッシュボード/アラート定義の変更が必要
移行: 段階導入、失敗時は旧定義へ戻す
```
