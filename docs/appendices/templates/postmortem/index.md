# ポストモーテムテンプレ

真因と再発防止を“行動”に落とし、フォローアップまで管理します。

## コピペ用テンプレ

```md
## 免責（責任追及しない）

- 個人の責任追及ではなく、仕組みの改善に焦点を当てる

## 事象の要約

- 何が起きたか
- 影響

## 真因（Root Cause）

- 直接原因
- 背後要因（プロセス/設計/運用）

## 再発防止（行動）

- 対策案（いつまでに/誰が/何を/どう検証するか）

## 検知改善

- より早く検知できたか

## 手順改善

- Runbook/手順書の更新点

## フォローアップ管理

- アクション一覧
- 期限とステータス
```

## 記入例（最小）

### 事象の要約

- 2026-02-23T10:12:00+09:00〜10:35、決済 API のタイムアウトが増加（最大 8% 失敗）

### 真因（Root Cause）

- 直接原因: DB 接続プール設定が負荷に耐えない
- 背後要因: 負荷試験不足、閾値の根拠が不明確

### 再発防止（行動）

- 対策: 接続プール設定の見直し + 負荷試験を追加
- 期限/担当/検証: 期限と担当を明記し、検証手順を PR に残す
